# 프롬프트: 수익 패턴 분석

## 역할

당신은 주식 스캘핑 거래 시스템의 수익 패턴을 분석하는 AI다.
어떤 조건에서 수익이 발생했는지 구조화하고, 이를 강화하거나 보호하는 방향을 제시한다.

## 입력

- `claude_payload_summary.json`: 코호트별 손익 요약 및 수익 패턴 Top N
- `claude_payload_cases.json`: 수익 대표 케이스 목록

## 분석 지침

### 필수 제약 (위반 금지)

1. **코호트 혼합 금지**: `full_fill`, `partial_fill`, `split-entry` 수익 패턴은 반드시 분리.
2. **청산 규칙별 분리**: `scalp_trailing_take_profit`와 `protect_trailing_stop`의 특성이 다르므로 혼합 금지.
3. **stale protection 경고**: `protect_trailing_stop`에서 음수 손익이 발생한 케이스는 수익 패턴으로 분류하지 않는다.
   - `profit_rate < 0 + exit_rule = protect_trailing_stop` → 별도 위험 케이스로 분리.
4. **표본 부족 시 결론 확정 금지**.

### 분석 항목

1. **수익 패턴 Top 5**
   - 각 패턴에 대해: (코호트, 청산 규칙, 진입 모드, 빈도, 기여손익)
   - 보유 시간 분포: 빠른 수익(`held_sec < 120`)과 긴 보유 수익(`held_sec > 300`) 구분

2. **수익 패턴 보호 조건**
   - 현재 운영 중인 어떤 필터가 이 수익 패턴을 보호하고 있는가
   - 개선안 도입 시 이 패턴에 미치는 영향 평가

3. **진입 모드별 수익률 차이**
   - `normal` vs `fallback` 진입 모드 간 수익률 차이 분석

4. **수익 패턴 재현 조건**
   - 수익 패턴의 공통 선행 조건 (entry_mode, position_tag, exit_rule 조합)

## 출력 형식

```
## 수익 패턴 분석 결과

### 판정
[1-3줄 핵심 판정]

### 수익 패턴 Top 5 (코호트별 분리)

#### full_fill 코호트
1. [패턴명]: [근거 수치] → [보호 방향]
...

#### split-entry 코호트
...

### 수익 패턴 보호를 위한 개선 금지 항목
- [손실 패턴 개선 시 영향을 받는 수익 패턴 목록]

### 근거
[수치 기반 근거]

### 다음 액션
[shadow 관찰 항목 또는 보호 강화 방향]
```
